# Portable Voice Markdown Export — Design Spec

**Date:** 2026-06-19
**Status:** Approved (design), pending implementation plan.
**Scope:** Add a "Download voice guide (.md)" export that renders the user's voice profile as a single, portable Markdown document — designed to paste into any external LLM (ChatGPT/Claude/Gemini) as a style instruction. Distinct from the existing ZIP *pack* export (which re-imports into BlogForge).

## Goal
- One click on the Voice screen downloads `*-voice-guide.md`.
- The document is self-contained and paste-ready: a short LLM-directed preamble, the persona, the distilled style, the user's banished list, and the **full** universal AI-tell lists.
- No new dependency; reuses `VoiceProfile` + `load_ai_tells()`.

## Decisions (locked)
- Content: my voice **+ the full AI-tell lists** (words/phrases/sentence-starters/patterns).
- Framing: **paste-ready LLM instruction** (a "When you write for me…" preamble) over readable sections.

## Architecture (3 pieces)

### 1 · `blogforge/voice/guide.py` — `build_voice_guide(profile: VoiceProfile) -> str`
Pure function (the testable core). Assembles Markdown from `profile` + `load_ai_tells()`:

```
# {profile.name} — Writing Voice Guide

> **How to use:** When you write for me, follow this voice guide — match the
> persona and style below, and never use the words, phrases, or patterns under
> "Avoid these AI-writing tells." Paste it into any AI assistant as a style
> instruction, or keep it as a personal reference.

## Persona
{persona_identity}

**In one line:** {persona_one_line}
**Tone:** {persona_tone}

## My style
{distilled_style_md}

## My banished words & phrases
Never use these in my writing:
- **Words:** {", ".join(rules.banished_words)}
- **Phrases:** {", ".join(rules.banished_phrases)}

## Avoid these AI-writing tells
Universal signs of machine-written text. Don't use them.

### Words to avoid
{", ".join(ai.words)}

### Phrases to avoid
{bullet list of ai.phrases}

### Sentence openers to avoid
{", ".join(ai.sentence_starters)}

### Structural patterns to avoid
{ai.patterns}    ← already Markdown bullets

---
*Generated by BlogForge from {len(profile.samples)} writing samples · distilled {distilled_at:%Y-%m-%d}.*
```

**Graceful omission (never crash on a sparse profile):**
- Persona fields that are empty are skipped (don't print empty "In one line:"/"Tone:").
- If `distilled_style_md` is empty → the "My style" section reads `*(Not yet distilled — run distillation on the Voice screen to capture your style.)*`.
- If both banished lists are empty → omit the "My banished words & phrases" section entirely.
- Footer date omitted if `distilled_at` is None; sample count always shown.

### 2 · Endpoint — `GET /api/voice/guide.md` (`api/voice.py`)
Next to `GET /api/voice/export`. Auth-gated (`Depends(get_current_user)`):
```python
@router.get("/guide.md")
async def export_guide(request: Request, current: User = Depends(get_current_user)) -> Response:
    store = _store(request)
    profile = await store.get_or_create(current.id)
    md = build_voice_guide(profile)
    slug = re.sub(r"[^a-z0-9]+", "-", (profile.name or "voice").lower()).strip("-") or "voice"
    return Response(
        content=md,
        media_type="text/markdown; charset=utf-8",
        headers={"Content-Disposition": f'attachment; filename="{slug}-voice-guide.md"'},
    )
```
No S3 access needed (unlike the ZIP export) — the guide is built purely from the profile row + bundled AI-tells.

### 3 · Frontend (`api/voice.ts`, `routes/VoicePage.tsx`)
- `api/voice.ts`: add `voiceGuideUrl()` returning `"/api/voice/guide.md"` (mirrors `voiceExportUrl`).
- `VoicePage.tsx`: add a second download anchor next to "Download pack":
  ```tsx
  <a href={voiceGuideUrl()} download className="nb-btn nb-btn-sm nb-btn-ghost shrink-0 mt-2">
    Download voice guide
  </a>
  ```
  (cookie auth carries through the `<a download>`, same as the ZIP button.)

## Testing
- **`build_voice_guide`** (unit): a profile with persona + `distilled_style_md` + banished words → assert the output contains the preamble ("When you write for me"), the persona identity, the distilled text, a banished word, and a known universal tell (`"delve"`) under "Words to avoid". A second test: an **empty** profile (no persona/distilled/banished) → no exception, the "not yet distilled" note appears, the banished section is absent, and the AI-tell lists are still present.
- **Endpoint** (`client` + signed cookie): `GET /api/voice/guide.md` → 200, `content-type` starts `text/markdown`, `Content-Disposition` is an attachment ending `-voice-guide.md`, body contains "Writing Voice Guide".
- Existing suite stays green; web `tsc` clean.

## Out of scope
- The ZIP pack export (unchanged).
- In-app editing/preview of the guide; per-section toggles; copy-to-clipboard (download only for v1).

## Success criteria
1. "Download voice guide" on the Voice screen downloads a `.md` with the persona, distilled style, the user's banished list, and the full universal AI-tell lists, under a paste-ready preamble.
2. A sparse/empty profile exports a valid guide (AI-tells + notes) without error.
3. New unit + endpoint tests pass; existing API suite + web `tsc` green.
